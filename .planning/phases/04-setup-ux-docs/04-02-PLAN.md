    # Jira 위젯 설정 가이드

    Waveterm의 Jira Tasks 위젯은 Atlassian Jira Cloud에서 본인에게 할당된 이슈를
    터미널 작업공간 안에서 바로 확인·분석할 수 있게 해줍니다. 이 문서는 처음
    설치한 팀원이 5분 안에 위젯에 자기 이슈가 뜨게 만드는 길을 안내합니다.

    ## Jira 위젯 소개

    - 본인에게 assign된 이슈 목록을 카드로 표시 (요약, 상태, 업데이트 시각, 댓글 수)
    - 카드 확장 시 description·attachments·comments 조회
    - `☁️` 버튼 한 번으로 Waveterm 백엔드(Go)가 Jira REST API를 호출해 캐시를 갱신
    - 분석 버튼은 선택한 CLI(`claude`, `gemini` 등)에 프롬프트를 전달
    - AI는 **선택**입니다. Jira 데이터 조회 자체는 AI 없이 동작합니다.

    ## 사전 준비

    - Atlassian 계정 (KakaoVX 내부 사이트: `https://<your-site>.atlassian.net`)
    - Jira Cloud 접근 권한 (JQL `assignee = currentUser()`로 조회 가능해야 함)
    - Waveterm v0.14.4 이상 (`wsh jira refresh` 명령이 포함된 빌드)

    ## PAT(API token) 발급

    1. 브라우저에서 <https://id.atlassian.com/manage-profile/security/api-tokens> 에 접속
    2. **Create API token** 클릭 → Label에 `waveterm-jira` 같이 알아볼 수 있는 이름 입력
    3. 생성된 토큰을 **즉시** 복사 (다시 표시되지 않습니다)
    4. 이 토큰이 아래 `jira.json`의 `apiToken` 값이 됩니다

    > 팁: PAT은 비밀번호 수준의 권한을 가집니다. `jira.json`은 권한 `0600`으로 저장하세요 (Windows는 사용자 홈 폴더 ACL로 보호됩니다).

    ## `jira.json` 작성

    `~/.config/waveterm/jira.json` 경로에 아래 템플릿을 붙여넣고 값만 바꾸세요.

    ```json
    {
      "baseUrl": "https://<YOUR_SITE>.atlassian.net",
      "cloudId": "<YOUR_CLOUD_ID>",
      "email": "<your@email.com>",
      "apiToken": "<PASTE_PAT_HERE>",
      "jql": "assignee = currentUser() ORDER BY updated DESC",
      "pageSize": 50
    }
    ```

    필드 설명:

    | 필드 | 설명 | 예시 |
    |---|---|---|
    | `baseUrl` | Atlassian 사이트 URL. 끝 슬래시 없이. | `https://kakaovx.atlassian.net` |
    | `cloudId` | 사이트의 고유 UUID. 아래 `## 검증` 바로 위의 `curl` 한 줄로 조회 가능. | `11111111-2222-3333-4444-555555555555` |
    | `email` | Atlassian 로그인 이메일 (Basic auth의 username) | `you@kakaovx.com` |
    | `apiToken` | 위에서 발급한 PAT | `ATATT3xFfGF0...` |
    | `jql` | 가져올 이슈를 선택하는 JQL. 기본값은 "내가 assign된 최근 업데이트 순". | `assignee = currentUser() ORDER BY updated DESC` |
    | `pageSize` | 한 번에 가져올 이슈 수 (최대 100) | `50` |

    ### cloudId를 어떻게 알아내나요?

    Atlassian 계정으로 브라우저에 로그인된 상태에서 아래 URL을 열면 JSON이 바로 반환됩니다:

    ```
    https://<your-site>.atlassian.net/_edge/tenant_info
    ```

    또는 터미널에서 (브라우저 세션 쿠키가 없어도 이 엔드포인트는 공개입니다):

    ```bash
    curl -s https://<your-site>.atlassian.net/_edge/tenant_info | jq -r .cloudId
    ```

    > 주의: `<your-site>`을 실제 사이트 이름으로 바꿔야 합니다 (예: `kakaovx`).

    ## 검증

    `jira.json`을 저장한 후 아무 터미널에서 다음을 실행하세요:

    ```bash
    wsh jira refresh
    ```

    성공하면 다음과 비슷한 출력이 나옵니다:

    ```
    ✔ Refreshed: 23 issues · 7 attachments · 41 comments · 1240ms
    cache: /Users/you/.config/waveterm/jira-cache.json
    ```

    이 시점에 Jira Tasks 위젯을 열거나 이미 열려 있다면 `☁️` 버튼을 눌러
    위젯이 캐시를 다시 읽게 하세요. 이슈 목록이 뜹니다.

    실패 시 위젯 카드와 `wsh jira refresh` CLI 둘 다 같은 한국어 메시지를
    보여줍니다 (예: `"인증 실패 — ..."`). 메시지별 대처는 § 문제 해결 참고.
