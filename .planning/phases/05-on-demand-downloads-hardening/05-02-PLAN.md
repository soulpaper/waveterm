**download.go** exports:

```go
// DownloadOpts is the input to DownloadAttachments.
type DownloadOpts struct {
    Config   Config
    IssueKey string // required, e.g. "ITSM-3135"
    Filename string // optional; "" = all attachments for the issue
}

// DownloadedFile describes a single successfully downloaded attachment.
type DownloadedFile struct {
    Filename  string `json:"filename"`
    Size      int64  `json:"size"`  // bytes written
    LocalPath string `json:"localpath"`
}

// DownloadReport summarizes the download operation.
type DownloadReport struct {
    IssueKey string           `json:"issuekey"`
    Files    []DownloadedFile `json:"files"`
    Errors   []string         `json:"errors,omitempty"` // per-file errors (non-fatal)
}

// DownloadAttachments downloads attachment(s) for the given issue.
func DownloadAttachments(ctx context.Context, opts DownloadOpts, hc *http.Client) (*DownloadReport, error)
```

**Implementation steps inside DownloadAttachments:**

1. Read cache via `cacheFilePath()` + `os.ReadFile` + `json.Unmarshal` into `JiraCache`.
2. Find the issue by `opts.IssueKey`. If not found, return error: `"issue %s not found in cache; run wsh jira refresh first"`.
3. Filter attachments: if `opts.Filename != ""`, match by filename (case-sensitive). If no match, return error listing available filenames.
4. Determine download directory: `attachmentDir(opts.IssueKey)` returns `~/.config/waveterm/jira-attachments/<KEY>/`. Call `os.MkdirAll`.
5. For each attachment to download:
   a. Build URL: `opts.Config.BaseUrl + "/rest/api/3/attachment/content/" + att.ID` (per D-DL-03).
   b. Create `http.NewRequestWithContext`, set Basic auth via a local helper (same as `setCommonHeaders` but just auth + accept). Actually, create a temporary `Client` and use `c.setCommonHeaders` — but `setCommonHeaders` is unexported. Instead, duplicate the 3 header lines inline (Authorization Basic, Accept, User-Agent). Keep it simple.
   c. `hc.Do(req)` — check response status.
   d. Create file: `os.Create(filepath.Join(dir, att.Filename))`.
   e. Stream: `io.Copy(file, resp.Body)`. Close both.
   f. Record `DownloadedFile{Filename, Size, LocalPath}`.
   g. On error: append to `report.Errors`, continue to next attachment (per D-DL-05 resilience).
6. After all downloads, update cache atomically (D-DL-04):
   a. Re-read cache (to pick up any concurrent changes from a parallel refresh).
   b. For each successfully downloaded file, find the matching attachment in cache by issue key + attachment ID, set `LocalPath`.
   c. `json.MarshalIndent` + `fileutil.AtomicWriteFile(cachePath, data, 0o600)`.
7. Return report.

**Helper function:**
```go
// attachmentDir returns ~/.config/waveterm/jira-attachments/<KEY>/
func attachmentDir(issueKey string) (string, error)
```

The function accepts `*http.Client` as a parameter (not creating its own) so that:
- Tests can inject httptest client
- Production code passes the same rate-limited/retry-wrapped client from NewClient
- Download requests automatically benefit from transport hardening (Plan 05-01/05-03)

For auth headers, create a small unexported helper:
```go
func setAuthHeaders(req *http.Request, cfg Config)
```
This duplicates the 3 lines from `setCommonHeaders` but is self-contained in download.go.
This avoids modifying client.go. Alternatively, export `setCommonHeaders` — but that changes
client.go which 05-03 also touches. Keep download.go self-contained.
