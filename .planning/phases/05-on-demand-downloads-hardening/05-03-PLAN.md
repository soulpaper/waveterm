**Modify NewClient in client.go:**

Replace the current `NewClient` implementation:

```go
// defaultRateLimit is the default rate limiter setting: 10 requests per second
// per Client instance (D-RL-01). Overridable via RefreshOpts.RateLimit.
const defaultRateLimit = 10.0

// defaultMaxRetries is the retry ceiling for transient failures (D-RETRY-03).
const defaultMaxRetries = 3

// NewClient constructs a Client with rate-limited, retry-capable transport.
// Transport wrapping order per D-RETRY-04:
//   http.DefaultTransport -> RateLimitedTransport(10 rps) -> RetryTransport(3 retries)
// Each retry attempt re-acquires a rate-limit token.
func NewClient(cfg Config) *Client {
    return NewClientWithRateLimit(cfg, defaultRateLimit)
}

// NewClientWithRateLimit constructs a Client with a custom rate limit.
// rps <= 0 disables rate limiting (D-RL-03). Retry is always enabled.
func NewClientWithRateLimit(cfg Config, rps float64) *Client {
    transport := NewRetryTransport(
        NewRateLimitedTransport(http.DefaultTransport, rps),
        defaultMaxRetries,
    )
    return &Client{
        cfg: cfg,
        hc:  &http.Client{Timeout: defaultTimeout, Transport: transport},
    }
}
```

`NewClientWithHTTP` remains UNCHANGED — it is the test seam where callers (tests, httptest)
provide their own http.Client with its own transport. This preserves backward compatibility
with all existing tests in client_test.go and refresh_test.go.

**Modify RefreshOpts in refresh.go:**

Add `RateLimit float64` field to `RefreshOpts`:

```go
// RateLimit overrides the default 10 req/s rate limiter (D-RL-03).
// 0 = no rate limiting. Negative values treated as 0.
// Only effective when HTTPClient is nil (otherwise the caller controls the transport).
RateLimit float64
```

In `Refresh()`, update the client construction block:

```go
var client *Client
if opts.HTTPClient != nil {
    client = NewClientWithHTTP(opts.Config, opts.HTTPClient)
} else if opts.RateLimit > 0 {
    client = NewClientWithRateLimit(opts.Config, opts.RateLimit)
} else if opts.RateLimit < 0 {
    // Explicitly disabled
    client = NewClientWithRateLimit(opts.Config, 0)
} else {
    // Default: 10 req/s
    client = NewClient(opts.Config)
}
```

This ensures:
- Existing tests passing `HTTPClient` still work (no transport wrapping applied).
- Production Refresh() calls get the default 10 req/s + 3 retries.
- Caller can override rate via `RateLimit` field.
