Create `pkg/jira/transport.go` in `package jira` with these exports:

**RateLimitedTransport:**
```go
// NewRateLimitedTransport wraps inner with a token-bucket rate limiter.
// rps = requests per second. If rps <= 0, no rate limiting is applied (passthrough).
func NewRateLimitedTransport(inner http.RoundTripper, rps float64) http.RoundTripper
```
- Struct holds `inner http.RoundTripper` and `limiter *rate.Limiter`.
- `RoundTrip`: call `limiter.Wait(req.Context())`, then `inner.RoundTrip(req)`.
- If `rps <= 0`, return inner directly (no wrapping). Per D-RL-03 this supports "0 = no limit".
- Burst = 1 per D-RL-01. Constructor: `rate.NewLimiter(rate.Limit(rps), 1)`.

**RetryTransport:**
```go
// NewRetryTransport wraps inner with retry logic for 429 and 5xx responses.
// maxRetries is the maximum number of retry attempts (not counting the initial request).
func NewRetryTransport(inner http.RoundTripper, maxRetries int) http.RoundTripper
```
- Struct holds `inner http.RoundTripper` and `maxRetries int`.
- `RoundTrip` logic:
  1. Execute `inner.RoundTrip(req)`.
  2. If response is 429:
     - Parse `Retry-After` via `parseRetryAfter(resp.Header.Get("Retry-After"))`.
     - If parsed > 60s, return response as-is (fatal per D-RETRY-02).
     - If parsed == 0, use 5s default (D-RETRY-02).
     - Sleep for the delay, respecting `req.Context()`. Use `select { case <-time.After(delay): case <-req.Context().Done(): return nil, req.Context().Err() }`.
     - Close the old response body before retrying (`resp.Body.Close()`).
     - Retry (loop back to step 1).
  3. If response is 5xx (500-599):
     - Compute backoff: `time.Duration(1<<attempt) * time.Second` giving 1s, 2s, 4s for attempts 0, 1, 2.
     - Sleep respecting context (same select pattern).
     - Close old response body before retrying.
     - Retry up to maxRetries.
  4. All other status codes: return response immediately (no retry).

**Wrapping order per D-RETRY-04:** `RetryTransport(RateLimitedTransport(http.DefaultTransport))`.
Each retry goes through the rate limiter because RetryTransport calls its inner (which is
RateLimitedTransport).

Do NOT add optional jitter (Claude's discretion: keep deterministic for testability).
Do NOT modify client.go in this plan — Plan 05-03 handles wiring.
